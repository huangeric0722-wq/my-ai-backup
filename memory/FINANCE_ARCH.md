# 財務科發展規劃 (FINANCE_ARCH.md)

## 1. AI 驅動的個人財務自動化研究

### 1.1 MCP 服務整合方案
經初步研究，針對 CSV/Excel 與圖片 OCR 的自動對帳，建議整合以下 open-source MCP servers：
- **Filesystem MCP**: 用於讀取本地 CSV/Excel 帳單檔案。
- **Google Drive MCP**: 若帳單存於雲端，可直接存取。
- **OCR 整合建議**: 
    - 目前尚無單一專用的「財務對帳 MCP」，建議開發一個中繼工具，結合 `pytesseract` (OCR) 與 `pandas` (數據對帳)。
    - 工具清單已列入 `learning_log.md`。

### 1.2 多維度帳戶預算分配策略 (專款專用)
利用 LLM 的推理能力進行「Eric 式專款專用分割」：
- **策略模型**: 建立 Prompt Template，輸入每月總收入與固定支出。
- **分割邏輯**: 
    - 生活支出 (Living)
    - 投資預備金 (Investment)
    - 緊急預備金 (Emergency)
    - 專項儲蓄 (Specific-purpose, e.g., 旅遊、大額採購)
- **自動化流**: LLM 接收對帳結果後，根據預設百分比或固定金額，產出「建議轉帳指令」或「預算報告」。

## 2. 30 萬投資監測系統規劃

### 2.1 自動化追蹤腳本
- **目標標的**: 華邦電 (2344)、南亞科 (2408)、聯昌 (2431)。
- **技術棧**: Python + `yfinance` 或 `FinMind`。
- **核心逻辑**: 
    - 每小時/每日爬取收盤價。
    - 比對歷史支撐位 (由 LLM 分析 K 線數據或設定固定值)。
    - 計算目前價格與「底部區間」的距離。

### 2.2 通知流程設計
1. **觸發條件**: 股價 <= 底部區間支撐位。
2. **通知渠道**: 
    - 使用 `message` 工具發送至 Discord/Telegram。
    - 本地日誌記錄。
3. **後續動作**: 通知內容需包含當前股價、支撐位價格、建議買入股數 (基於 30 萬總額比例)。
